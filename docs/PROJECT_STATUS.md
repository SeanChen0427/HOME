## 四、已確認的品牌與課程定位

Sean 的優勢來自大量第一線社群操作經驗。他知道業主日常真正會使用的
技巧與情境，再將這些工作拆成容易吸收、可以實際操作的課程。

主要教學特色：

- 從業主的實際工作場景出發，不照軟體功能表授課。
- 相較八小時塞入大量內容，更偏好一次聚焦一個主軸。
- 課程會保留操作時間，讓學員不只聽懂，也能開始動手。
- AI 經常自然融入社群與行銷課程，不為了標榜 AI 而硬塞工具。
- 企業 AI 專班從有效溝通及工作需求出發，依任務選擇不同 AI。
- 不將課程綁死在 ChatGPT；不同工具有各自適合的工作。

## 五、已確認的 UI/UX 決策

### 版面

- 避免大量等尺寸方格或卡片牆，資訊會顯得擁擠且難以閱讀。
- 邀約頁採編輯式長頁，以留白、細線及文字層級建立閱讀節奏。
- 課程 1–6 使用緊湊目錄形式，不使用固定高度。
- 手機版形象照使用小型橫向講師卡，不可讓人像占滿多個螢幕。
- 標題大小應克制，不可大到產生壓迫感。
- 手機版是每次修改後的必要驗收項目。

### 中文換行

- 禁止只為視覺平均使用 `text-wrap: balance` 切斷中文語意。
- 禁止出現「一件／事」、「而是／確保」等不自然斷句。
- 重要標題在 JSX 中以 `.title-phrase` 人工標示不可拆分的語意片語。
- `.title-phrase` 使用 `white-space: nowrap`，只允許在片語邊界換行。
- 一般內文使用 `line-break: strict`、`word-break: normal` 與
  `text-wrap: pretty`。
- 修改標題文字時，必須同步檢查人工片語是否仍符合新文案。

## 九、修改與驗收方式

修改內容後至少執行：

```bash
npm run build
npm run lint
```

已知情況：ESLint 偶爾會長時間沒有輸出。不要因此重複啟動多個 lint 程序；
先等待，必要時確認既有程序狀態後再重新執行。

前端修改後必須實際檢查：

- 桌機寬度
- 約 390px 的手機寬度
- 是否水平溢位
- 中文標題是否依語意換行
- 人像、按鈕及段落是否占用過多首屏空間
